프로젝트 개발 순서

[SpringMVC]
SQL(mapper) -> DAO/DAOImpl -> Service/ServiceImpl -> Controller -> View

[SpringBoard]
Controller -> View -> Controller -> Service/ServiceImpl -> DAO/DAOImpl -> SQL(mapper)


	* Controller 준비
	- 공통적인 URI경로 설정(컨트롤러를 구분 목적)
	- 각 기능별 URI를 설정(+ 호출방식 GET/POST)
	- 연결된 뷰페이지 처리(정보전달Model / 페이지이동 redirect)
	
	* Service 준비
	- 컨트롤러와 DAO를 연결 기능을 설정

	* DAO 준비
	- DB 연결 정보 처리 설정(root-context.xml)
	- Mysql에 데이터베이스/테이블 생성
	- SQL 구문(mapper) 준비


	* DB생성
	
	CREATE TABLE `springdb`.`tbl_board` (
  `bno` INT NOT NULL AUTO_INCREMENT COMMENT '게시판의 글번호를 설정\nAI (자동으로 숫자를 증가시켜가면서 설정)',
  `title` VARCHAR(200) NOT NULL COMMENT '제목',
  `content` TEXT NULL COMMENT '내용',
  `writer` VARCHAR(50) NOT NULL COMMENT '글쓴이',
  `regdate` TIMESTAMP NOT NULL DEFAULT now() COMMENT '작성날짜',
  `viewcnt` INT NULL DEFAULT 0 COMMENT '글 조회수(기본값은 0)',
  PRIMARY KEY (`bno`))
COMMENT = '게시판 정보를 저장하는 테이블';


  * 게시판 만들기 프로젝트
  - 컨트롤러는 /board/* 로 매핑한다

  - 기능정의 
  * 동작에 따른 처리방식을 결정
    GET 방식 : 조회 / 출력 / 입력 동작 수행시 
    POST 방식 : 처리(등록, 수정, 삭제...) 동작 수행시
  
  - 멤버 : 회원가입 -> 게시판 : 글쓰기
  
  - 게시판 글쓰기
    1) 사용자가 글정보를 입력하는 페이지 / GET / 주소 : /board/regist
    2) 입력된 글정보를 처리하는 페이지 / POST / 주소 : /board/regist
    
  - 게시판 목록 보기(조회)
  	1) 컨트롤러에 리스트 출력하는 동작 페이지 생성

  	2) 서비스 -> DB 데이터 가져오기

  	3) 연결된 뷰페이지에 출력(+ 템플릿)
  
  - 게시판 본문(내용)보기 
  - 게시판 내용 수정
  - 게시판 내용 삭제

  * 컨트롤러의 정보를 연결된 뷰페이지에 전달
  
  Model 객체 (택배 기사님)
   : 스프링 제공 객체(컨트롤러의 데이터를 뷰페이지로 전달)
    addAttribute("이름", 값); => 주소줄에 정보 전달 & 정보 유지
  
  RedirectAttirbute 객체 (쿠팡 기사님)
   : 스프링 제공 객체(컨트롤러의 데이터를 뷰페이지로 전달)
      단, 페이지를 리다이렉트 이동 시 사용가능
      
    addAttribute("이름", 값); => 주소줄에 정보 전달 & 정보 유지
    addFlashAttribute("이름", 값); => 주소줄에 정보 전달 X (세션에 저장)& 정보 유지 X
    												(새로고침 시 사라짐)
    
    
    @ModelAttribute("값이름") 저장할 변수
  		: 전달된 데이터(파라메터)를 저장방법
  		
  		영역객체.getAttribute(); 동작처럼 수행가능
  		기본적인 형변환이 가능
  		1:N 관계에서 사용
  		- 컨트롤러에 생성된 model 객체를 사용해서 저장후 전달
  		  model.addAttribute("이름",값); 동작을 처리함
  		  
  
  	@RequestParam("값이름") 저장할 변수
  		: 전달된 데이터(파라메터)를 저장방법
		
		request.getParamater(); 동작처럼 수행가능
		기본적인 형변환이 가능
		* 1:1 대응 관계
